# OpenVPN Traffic Monitor

### Project Description:

**English:**
This application is designed to monitor and analyze the traffic of OpenVPN clients. It automatically parses OpenVPN logs, extracts information about client connections, including received and sent traffic, and then stores this data in an SQLite database. The system tracks the traffic of individual clients, and also keeps records of traffic usage by month.

**Russian:**
Это приложение предназначено для мониторинга и анализа трафика клиентов OpenVPN. Оно автоматически разбирает логи OpenVPN, извлекает информацию о подключениях клиентов, включая полученный и отправленный трафик, а затем сохраняет эти данные в базе данных SQLite. Система отслеживает трафик отдельных клиентов, а также ведет учет использования трафика по месяцам.

### Installation

**English:**
1.  **Clone the Repository:**
    ```bash
    git clone https://github.com/baffoRti/OpenVPN_Traffic_Monitor
    ```

2.  **Set up OpenVPN Status Log:**
    Ensure your OpenVPN server is configured to export its status log. To do this, check the file `/etc/openvpn/ovpn.conf`. There should be lines:
    ```
    status /var/log/openvpn/status.log
    status-version 2
    ```
    If there are no such lines, then add. If you plan to use a different path, do not forget to specify it in the file `config.py `.

4.  **Run the Main Application (Scheduled):**
    To process logs and update the database automatically, set up a cron job. Make the script `OpenVPN_Traffic_Monitor` executable (replace `/path/to/your/project/` with your actual project path):
    ```bash
    chmod +x /path/to/your/project/OpenVPN_Traffic_Monitor.sh
    ```
    Then, add it to your crontab to run every minute:
    ```bash
    crontab -e
    ```
    Add the following line:
    ```cron
    */n * * * * /path/to/your/project/OpenVPN_Traffic_Monitor.sh
    ```
    where n is the start frequency in minutes (recommended value from 1 to 5)

**Russian:**
1.  **Клонирование репозитория:**
    ```bash
    git clone https://github.com/baffoRti/OpenVPN_Traffic_Monitor
    ```

2.  **Настройте лог статуса OpenVPN:**
    Убедитесь, что ваш сервер OpenVPN настроен на экспорт лога статуса. Для этого проверьте файл `/etc/openvpn/ovpn.conf`. Там должны быть строки:
    ```
    status /var/log/openvpn/status.log
    status-version 2
    ```
    Если таких строк нет, то добавьте. Если планируете использовать другой путь, не забудьте указать его в файле `config.py`.

4.  **Запуск основного приложения (по расписанию):**
    Для автоматической обработки логов и обновления базы данных настройте cron-задачу. Сделайте скрипт `OpenVPN_Traffic_Monitor` исполняемым (замените `/путь/к/вашему/проекту/` на ваш фактический путь к проекту):
    ```bash
    chmod +x /путь/к/вашему/проекту/OpenVPN_Traffic_Monitor.sh
    ```
    Затем добавьте его в crontab для выполнения каждую минуту:
    ```bash
    crontab -e
    ```
    Добавьте следующую строку:
    ```cron
    */n * * * * /путь/к/вашему/проекту/OpenVPN_Traffic_Monitor.sh
    ```
    где n - частота запуска в минутах (рекомендуемое значение от 1 до 5)
