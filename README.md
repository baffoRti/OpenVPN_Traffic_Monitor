# OpenVPNStats

An application for collecting statistics of used OpenVPN traffic



# OpenVPN Traffic Monitor

### Project Description:

**English:**
This project is designed to monitor and analyze OpenVPN client traffic. It automatically parses OpenVPN status logs, extracts client connection details, including bytes received and sent, and then stores this data in a structured SQLite database. The system tracks individual client traffic, differentiates between new and continuing sessions, and records historical monthly usage, providing valuable insights into VPN client activity and data consumption.

**Russian:**
Этот проект предназначен для мониторинга и анализа трафика клиентов OpenVPN. Он автоматически разбирает логи статуса OpenVPN, извлекает детали подключений клиентов, включая полученные и отправленные байты, а затем сохраняет эти данные в структурированной базе данных SQLite. Система отслеживает трафик отдельных клиентов, различает новые и продолжающиеся сессии, а также записывает исторические данные ежемесячного использования, предоставляя ценную информацию об активности клиентов VPN и потреблении данных.

### Installation

**English:**
1.  **Clone the Repository:**
    ```bash
    git clone http://example.com/openvpn-traffic-monitor.git
    cd openvpn-traffic-monitor
    ```

2.  **Set up OpenVPN Status Log:**
    Ensure your OpenVPN server is configured to export its status log to `/content/status_example.log` (or update `config.py` with the correct path).

3.  **Run the Main Application (Scheduled):**
    To process logs and update the database automatically, set up a cron job. First, create a wrapper script, e.g., `run_monitor.sh`:
    ```bash
    #!/bin/bash
    /usr/bin/python3 /path/to/your/project/main.py >> /path/to/your/project/openvpn_stats.log 2>&1
    ```
    Make the script executable:
    ```bash
    chmod +x /path/to/your/project/run_monitor.sh
    ```
    Then, add it to your crontab to run every minute:
    ```bash
    crontab -e
    ```
    Add the following line (replace `/path/to/your/project/` with your actual project path):
    ```cron
    * * * * * /path/to/your/project/run_monitor.sh
    ```

**Russian:**
1.  **Клонирование репозитория:**
    ```bash
    git clone http://example.com/openvpn-traffic-monitor.git
    cd openvpn-traffic-monitor
    ```

2.  **Настройте лог статуса OpenVPN:**
    Убедитесь, что ваш сервер OpenVPN настроен на экспорт лога статуса в `/content/status_example.log` (или обновите `config.py` с правильным путем).

3.  **Запуск основного приложения (по расписанию):**
    Для автоматической обработки логов и обновления базы данных настройте cron-задачу. Сначала создайте скрипт-обертку, например, `run_monitor.sh`:
    ```bash
    #!/bin/bash
    /usr/bin/python3 /путь/к/вашему/проекту/main.py >> /путь/к/вашему/проекту/openvpn_stats.log 2>&1
    ```
    Сделайте скрипт исполняемым:
    ```bash
    chmod +x /путь/к/вашему/проекту/run_monitor.sh
    ```
    Затем добавьте его в crontab для выполнения каждую минуту:
    ```bash
    crontab -e
    ```
    Добавьте следующую строку (замените `/путь/к/вашему/проекту/` на ваш фактический путь к проекту):
    ```cron
    * * * * * /путь/к/вашему/проекту/run_monitor.sh
    ```
