# OpenVPNStats

An application for collecting statistics of used OpenVPN traffic



\# OpenVPN Traffic Monitor



\### Project Description:



\*\*English:\*\*

This project is designed to monitor and analyze OpenVPN client traffic. It automatically parses OpenVPN status logs, extracts client connection details, including bytes received and sent, and then stores this data in a structured SQLite database. The system tracks individual client traffic, differentiates between new and continuing sessions, and records historical monthly usage, providing valuable insights into VPN client activity and data consumption.



\*\*Russian:\*\*

Этот проект предназначен для мониторинга и анализа трафика клиентов OpenVPN. Он автоматически разбирает логи статуса OpenVPN, извлекает детали подключений клиентов, включая полученные и отправленные байты, а затем сохраняет эти данные в структурированной базе данных SQLite. Система отслеживает трафик отдельных клиентов, различает новые и продолжающиеся сессии, а также записывает исторические данные ежемесячного использования, предоставляя ценную информацию об активности клиентов VPN и потреблении данных.

\### Installation



\*\*English:\*\*

1\.  \*\*Clone the Repository:\*\*

&#x20;   ```bash

&#x20;   git clone http://example.com/openvpn-traffic-monitor.git

&#x20;   cd openvpn-traffic-monitor

&#x20;   ```



2\.  \*\*Set up OpenVPN Status Log:\*\*

&#x20;   Ensure your OpenVPN server is configured to export its status log to `/content/status\_example.log` (or update `config.py` with the correct path).



3\.  \*\*Run the Main Application (Scheduled):\*\*

&#x20;   To process logs and update the database automatically, set up a cron job. First, create a wrapper script, e.g., `run\_monitor.sh`:

&#x20;   ```bash

&#x20;   #!/bin/bash

&#x20;   /usr/bin/python3 /path/to/your/project/main.py >> /path/to/your/project/openvpn\_stats.log 2>\&1

&#x20;   ```

&#x20;   Make the script executable:

&#x20;   ```bash

&#x20;   chmod +x /path/to/your/project/run\_monitor.sh

&#x20;   ```

&#x20;   Then, add it to your crontab to run every minute:

&#x20;   ```bash

&#x20;   crontab -e

&#x20;   ```

&#x20;   Add the following line (replace `/path/to/your/project/` with your actual project path):

&#x20;   ```cron

&#x20;   \* \* \* \* \* /path/to/your/project/run\_monitor.sh

&#x20;   ```
